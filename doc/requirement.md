# VodiceBook 需求文档

> 小说转有声书平台（MVP 阶段需求已全部实现，详见 [mvp.md](mvp.md)）

## 1. 项目背景

随着有声书市场的快速增长，传统人工录制有声书面临**成本高、周期长、产能有限**的瓶颈。AI 语音合成技术的成熟使得自动化生产有声书成为可能。

VodiceBook 旨在构建一套 **AI 驱动的小说转有声书系统**，用户上传小说章节文本后，系统自动完成角色识别、朗读稿生成、多角色语音合成，最终输出高质量的有声书音频。

## 2. 目标用户

| 用户角色 | 描述 |
|---------|------|
| 内容创作者 | 网文作者、独立出版人，希望将文字作品转化为有声书 |
| 有声书制作方 | 小型工作室，需要降低制作成本和缩短制作周期 |
| 个人用户 | 希望将喜欢的小说转为音频收听的普通读者 |

## 3. 核心需求

### 3.1 章节文本上传

- **FR-001**: 用户能够上传小说某一章节的文本内容
- **FR-002**: 系统支持纯文本（.txt）格式的输入
- **FR-003**: 系统应对输入文本进行基本校验（编码检测、空内容检查、长度限制）

### 3.2 朗读稿生成（LLM 层）

- **FR-010**: 系统调用大语言模型（LLM）分析小说文本，识别以下要素：
  - 旁白/叙述部分
  - 对话部分及对应的说话角色
  - 角色的情感状态（喜、怒、哀、惧等）
- **FR-011**: 系统基于分析结果生成结构化的**朗读稿**，朗读稿应包含：
  - 分段信息（叙述段 / 对话段）
  - 每段对应的角色标识
  - 情感/语调标注
  - 原始文本内容
- **FR-012**: 朗读稿应以结构化格式（如 JSON）存储，便于下游 TTS 模块消费
- **FR-013**: 用户可以在生成后对朗读稿进行预览和微调（如修改角色分配、情感标注）

### 3.3 语音合成（TTS 层）

- **FR-020**: 系统根据朗读稿，按段落逐一调用 TTS 模型生成语音
- **FR-021**: 不同角色应使用不同的音色/声线，以区分角色
- **FR-022**: 旁白部分使用专用的叙述者音色
- **FR-023**: 语音合成应尊重情感标注，在语调上体现情绪变化
- **FR-024**: 系统将多段语音按顺序拼接为完整的章节音频
- **FR-025**: 最终输出的音频格式为常用格式（如 MP3、WAV）

### 3.4 音频管理

- **FR-030**: 用户能够在线试听生成的音频
- **FR-031**: 用户能够下载生成的音频文件
- **FR-032**: 系统保留生成历史记录，用户可查看和重新下载

## 4. 非功能性需求

### 4.1 性能

- **NFR-001**: 单章节（约 5000 字）的完整处理时间不超过 5 分钟（LLM 分析 + TTS 合成）
- **NFR-002**: 系统应支持异步任务处理，用户提交后无需等待，可查看任务进度

### 4.2 可用性

- **NFR-010**: 提供简洁直观的 Web 操作界面
- **NFR-011**: 操作流程不超过 3 步：上传 → 确认朗读稿 → 生成音频

### 4.3 可扩展性

- **NFR-020**: LLM 和 TTS 模型应通过配置切换，不与特定供应商强耦合
- **NFR-021**: 音色库应支持扩展，用户可新增自定义角色音色

### 4.4 安全性

- **NFR-030**: API Key 等敏感信息不得硬编码在代码中
- **NFR-031**: 用户上传的文本内容需进行基本的内容安全校验

## 5. 技术约束

| 约束项 | 说明 | MVP 状态 |
|-------|------|----------|
| LLM 服务 | 通过 SiliconFlow 平台调用 Qwen2.5-7B-Instruct | ✅ 已实现 |
| TTS 服务 | 阿里云 DashScope qwen3-tts-flash | ✅ 已实现 |
| 部署环境 | 需兼容低配服务器（4核 4GB 内存），核心计算通过 API 外包 | ✅ 已验证 |
| 技术栈 | Spring Boot 3.2.5 + Java 21 + H2 + Thymeleaf | ✅ 已实现 |

## 6. 项目边界

### 在范围内（In Scope）

- 单章节文本的上传与处理
- LLM 驱动的角色识别与朗读稿生成
- TTS 多角色语音合成与音频拼接
- 基础的 Web 操作界面
- 任务状态查询与音频下载

### 不在范围内（Out of Scope）

- 整本小说的批量处理（后续迭代）
- 用户自训练音色模型（后续迭代）
- 实时流式语音生成
- 多语言支持（当前仅中文）
- 移动端应用

## 7. 术语表

| 术语 | 定义 |
|------|------|
| 朗读稿 | 由 LLM 分析小说文本后生成的结构化中间产物，包含角色、情感、分段等标注信息 |
| TTS | Text-to-Speech，文本转语音技术 |
| LLM | Large Language Model，大语言模型 |
| SiliconFlow | 国内 AI 模型 API 聚合平台，提供多种模型的统一调用接口 |
| QwenTTS | 阿里通义千问团队的语音合成模型 |
